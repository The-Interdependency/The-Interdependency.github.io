Projects and consolidation plan

This page consolidates what is publicly visible across the website repo, The-Interdependency/a0, and erinepshovel-code/aimmh. It is written to reduce overclaiming and to make overlap legible.

Repository snapshots

  • wayseer00/wayseer.github.io: static website repo with HTML/CSS pages, canon materials, support page, and GitHub Pages deployment.
  • The-Interdependency/a0: larger application repo with React/TypeScript frontend, Express middle layer, Python/FastAPI services, PostgreSQL-oriented configuration, and Cloud Run deployment materials.
  • erinepshovel-code/aimmh: multi-model AI hub with React frontend, FastAPI backend, MongoDB-oriented services, and a reusable aimmh_lib orchestration library.

Overlap that should be named clearly

  • Both a0 and aimmh describe multi-model or agent-oriented AI application work.
  • Both contain React frontends and Python/FastAPI backends.
  • Both include product-language elements such as pricing, auth, and deployment-related material.
  • The website currently mixes canon language, software language, and demonstration language; these should be separated more cleanly.

Suggested consolidation structure

  1. Website repo: keep as the public front door, canon index, bounded project summaries, and support links.
  2. a0 repo: present as the primary agent-platform codebase if that remains the main long-term direction.
  3. aimmh repo: present as either a sibling product, an earlier branch of work, or a modular source of reusable orchestration patterns.
  4. Shared language: define one short glossary for terms that recur across repos.

Practical next moves

  1. Remove or rewrite pages that state deployment, consciousness, or capability as settled fact when the page is really a note.
  2. Create one repo-to-purpose map so visitors can tell what each repository is for in under a minute.
  3. Move reusable technical patterns into explicit shared docs or libraries instead of repeating narrative claims across pages.
  4. Keep the website minimal: canon, projects, notes, pictures, support.

Investigation notes

  • Sub-agent review: a0 surfaced a complex application stack with deployment docs and multiple runtime layers.
  • Sub-agent review: aimmh surfaced a more explicit multi-model AI hub narrative plus an extracted library suitable for reuse.
  • Website review surfaced stale links and pages whose tone exceeded what the public repos alone can support.
