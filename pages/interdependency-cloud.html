Interdependency Cloud

Official Node Deployment Guide — How to bring up the first living compute nodes of the Interdependent Way.

Phase 0: Authenticate to Google Cloud

gcloud auth login erin.eps.hovel@gmail.com
gcloud config set project wayseer-interdependent-cloud

gcloud services enable compute.googleapis.com iam.googleapis.com cloudresourcemanager.googleapis.com logging.googleapis.com monitoring.googleapis.com oslogin.googleapis.com

Phase 1: Create the VM (recommended specs)

gcloud compute instances create interdependent-node-1 \\
  --zone=us-central1-a \\
  --machine-type=e2-standard-4 \\
  --boot-disk-size=100GB \\
  --image-family=ubuntu-2204-lts \\
  --image-project=ubuntu-os-cloud \\
  --scopes=default,cloud-platform \\
  --tags=http-server,https-server

Phase 2: Get IP and SSH in

gcloud compute instances describe interdependent-node-1 --zone=us-central1-a --format='get(networkInterfaces[0].accessConfigs[0].natIP)'

gcloud compute ssh interdependent-node-1 --zone=us-central1-a

Phase 3: Run these commands inside the VM (copy-paste block)

sudo apt update && sudo apt upgrade -y
sudo apt install -y git curl nginx certbot python3-certbot-nginx htop unzip

sudo mkdir -p /opt/interdependent/{bin,logs,config,modules,repo}
sudo chown -R $USER:$USER /opt/interdependent

cd /opt/interdependent
git clone https://github.com/wayseer00/wayseer.github.io.git repo

cd repo
git config --global user.name "Way Seer"
git config --global user.email "erin.eps.hovel@gmail.com"

echo "✅ Node connected to GitHub successfully."
pwd && ls -la
